# Tone Correction Agent (어투 교정 에이전트)

## 어떤 병목을 다루는가
- **Task**: 이메일, 보고서, 발표 자료 작성 시 문장 표현 다듬기
- **빈도**: 일 3~5회 (업무 및 학업 시 상시)
- **시간**: 문장 하나를 다듬는 데 평균 1 interval (10~30분) 소비
- **왜 병목인가**: 단순한 오타 수정이 아니라, 상황(비즈니스, 학술, 일상)에 맞는 적절한 어휘 선택과 문맥 파악에 많은 에너지가 소모됨. 특히 외국어나 격식을 차려야 하는 상황에서 심리적 부담과 시간 지연이 큼.

## 왜 AI Agent로 만들었는가
- **룰베이스가 안 되는 이유**: "문장 끝을 ~합니다로 바꿔라" 정도의 규칙은 가능하지만, 단어의 미묘한 뉘앙스 차이(뉘앙스, 어감)나 문맥에 따른 적합성 판단은 단순 치환으로 불가능함.
- **AI 판단이 필요한 지점**: 
  - 입력된 문장의 원래 의도 파악
  - 선택된 '톤'에 가장 어울리는 어휘 추천
  - 자연스러운 문장 흐름(Flow) 생성
  - 왜 그렇게 수정했는지에 대한 논리적 근거 제시

## Agent 구조
- **입력**: 원문 텍스트, 대상 톤(공손, 학술, 캐주얼 등)
- **처리**: 
  1. 입력 유효성 검사 (너무 짧거나 불분명하면 반려)
  2. 문맥 분석 및 톤 변환 수행
  3. 변환 결과와 함께 '변경 이유' 생성
  4. 체크리스트 준수 여부 자가 검증
- **출력**: 수정된 문장, 변경 이유(Reasoning)
- **핵심 제약**: 원문 의미 유지, 문장 길이 ±20% 유지, 톤 일관성

## 실행 방법
1. 원문 텍스트를 입력창에 넣습니다.
2. 원하는 페르소나/톤을 선택합니다.
3. '변환하기' 버튼을 클릭합니다.
4. 결과물과 상세 리즈닝을 확인합니다.

## 테스트 입력 형식
- `test-input/` 폴더에 JSON 형식으로 저장됨.
- `text`: 원문 문장
- `targetTone`: 목표 톤

## 실행 결과 (5회 일관성 테스트)
- 테스트 입력: "나 이거 다 했어. 언제 줄까?" (Target: Polite)
- 필드: `originalText`, `correctedText`, `reasoning`, `toneMatch`, `lengthConstraint`
- **Result 1-5**: 일관되게 공손한 표현("~을 완료하였습니다", "~전달해 드릴까요?")으로 변환됨.
- 상세 결과는 `CHECKLIST.md` 및 앱 내 히스토리 참조.
