# SPEC: 무엇·왜 (Requirements)

## 1. 프로젝트 개요
사용자가 작성한 초안 텍스트를 특정 상황(공손, 학술, 캐주얼 등)에 맞는 최적의 어투로 변환해주는 AI 에이전트.

## 2. 해결하고자 하는 문제 (Bottleneck)
- **병목 지점**: 문장 작성 후 "이게 너무 예의 없어 보이지 않나?", "너무 구어체인가?" 하는 고민에 빠져 수차례 퇴고를 반복함.
- **수치 근거**: 1개 이메일 작성 시 표현 교정에만 약 15분 소요. 주 10회 이상 발생. 
- **실패 비용**: 격식에 맞지 않는 표현으로 인한 오해 발생, 신뢰도 하락.

## 3. 핵심 요구사항
- **의미 보존**: 표현은 바뀌어도 사용자가 전달하고자 하는 핵심 메시지는 변하면 안 됨.
- **톤 변환**: 선택한 톤(Polite, Academic, Business, Casual)에 명확하게 부합해야 함.
- **논리적 근거**: 단순 결과만 주는 것이 아니라, "공손함을 위해 ~습니다체를 사용했고, ~라는 단어를 선택했습니다"와 같은 설명 제공.
- **입력 검증**: 5글자 미만의 입력이나 의미 없는 텍스트는 반려(Rejection) 로직 필요.
- **길이 제약**: 원문 대비 길이 변화가 ±20% 이내여야 함 (간결성 유지).

## 4. 수용 기준 (Acceptance Criteria)
- [ ] 입력이 5자 이상인가?
- [ ] 출력된 문장이 선택한 톤을 유지하는가?
- [ ] 변경 이유가 포함되어 있는가?
- [ ] 원문과 의미가 95% 이상 일치하는가?
- [ ] 문장 길이가 원문의 80%~120% 사이인가?
